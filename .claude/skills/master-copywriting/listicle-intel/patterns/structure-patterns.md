# Listicle Structure Patterns

## Purpose

Proven structural patterns for high-converting listicles. Structure affects both SEO and conversion.

---

## Core Structure (Required)

Every listicle MUST have these sections in order:

```
1. Hook Opening (2-4 sentences)
2. Value Summary (1 paragraph)
3. Qualification Section
4. Comparison Content (main body)
5. Strategic Offer Placements (3-4)
6. Interactive CTA Elements
7. Clear Next Steps / FAQ
```

---

## Section-by-Section Patterns

### 1. Hook Opening Patterns

**Pattern A: Pain Acknowledgment**
```
[Pain point statement]. [Amplification]. [Promise of solution].

Example:
"Drowning in credit card debt? You're not alone - millions of Americans
are paying more in interest than they need to. But there's a way out."
```

**Pattern B: Surprising Statistic**
```
[Statistic]. [Why it matters]. [What this means for reader].

Example:
"The average American pays $1,200 more per year on [service] than necessary.
That's money that could be [benefit]. Here's how to keep more of it."
```

**Pattern C: Direct Question**
```
[Question addressing intent]. [Validation]. [Promise].

Example:
"Looking for the best [category] options? You've come to the right place.
We've analyzed [X] providers to help you find the perfect fit."
```

---

### 2. Value Summary Patterns

**Pattern A: What You'll Learn**
```
In this guide, you'll discover:
- [Benefit 1]
- [Benefit 2]
- [Benefit 3]

Plus, [bonus value].
```

**Pattern B: Time Investment Justification**
```
In the next [X] minutes, you'll have everything you need to [achieve goal].
We've done the research so you don't have to.
```

---

### 3. Qualification Section Patterns

**Pattern A: Who This Is For**
```
This guide is for you if:
- [Situation 1]
- [Situation 2]
- [Situation 3]

If [disqualifying factor], see [alternative resource].
```

**Pattern B: Quick Assessment**
```
Before we dive in, a quick check:
- [ ] [Qualifying question 1]
- [ ] [Qualifying question 2]

If you checked [X]+, keep reading.
```

---

### 4. Comparison Content Patterns

**Pattern A: Card Layout**
```
## [Option Name]
[1-2 sentence summary]

**Best For:** [Audience/situation]
**Key Features:**
- [Feature 1]
- [Feature 2]
- [Feature 3]

**Pros:** [Quick list]
**Cons:** [Honest list]

[CTA Button]
```

**Pattern B: Table Comparison**
```
| Feature | Option A | Option B | Option C |
|---------|----------|----------|----------|
| Price   | $X       | $Y       | $Z       |
| [Key 1] | Yes      | No       | Yes      |
| [Key 2] | [value]  | [value]  | [value]  |

**Our Pick:** [Option] because [reason]
```

**Pattern C: Ranked List**
```
### #1: [Option Name] - Best Overall
[Why it's #1]

### #2: [Option Name] - Best for [Use Case]
[Why it's good for this]

### #3: [Option Name] - Best Budget Option
[Value proposition]
```

---

### 5. Offer Placement Patterns

**Placement Strategy:**
```
Position 1: After qualification section (warm leads)
Position 2: After first 2-3 comparisons (engaged readers)
Position 3: After addressing objections (ready to act)
Position 4: Final comprehensive CTA (decision time)
```

**Offer Box Pattern:**
```
┌─────────────────────────────────────────┐
│ [Headline matching reader stage]        │
│                                         │
│ [1-2 sentences of value]                │
│                                         │
│ [Interactive Element / Input]           │
│                                         │
│ [CTA Button]                            │
│ [Subtext: friction reducer]             │
└─────────────────────────────────────────┘
```

---

### 6. Interactive CTA Patterns

**Pattern A: Amount Selector**
```
How much [quantity]?
[Slider or Dropdown]
[CTA: See My Options]
```

**Pattern B: Situation Quiz**
```
What best describes you?
[ ] [Option A]
[ ] [Option B]
[ ] [Option C]
[CTA: Get Matched]
```

**Pattern C: ZIP Entry**
```
Find [service] near you:
[ZIP Input Field]
[CTA: Show Local Options]
```

---

### 7. Closing Section Patterns

**Pattern A: FAQ**
```
## Frequently Asked Questions

### [Common Question 1]
[Answer - 2-3 sentences]

### [Common Question 2]
[Answer - 2-3 sentences]

[Final CTA]
```

**Pattern B: Next Steps**
```
## What to Do Next

1. [First action] - [Why]
2. [Second action] - [What happens]
3. [Third action] - [Result]

[Final CTA with urgency]
```

## Mobile Structure Adaptations

### Mobile-First Rules:
- Max 3 sentences per paragraph
- Headers every 200-300 words
- CTAs visible without scrolling past fold
- Tables converted to cards on mobile
- Interactive elements thumb-friendly

---

## SEO Structure Requirements

### For Search Performance:
- H1: Main keyword + benefit
- H2s: Question-format keywords
- H3s: Option/feature names
- FAQ schema for featured snippets
- Table of contents for long pieces
